# myDSAproject

Current project structure:
- scripts/eda_first_pass.py -> first-pass EDA, same-day correlations, and lag correlation screening
- scripts/modeling_baseline.py -> baseline occupancy forecasting models with and without lagged Google Trends features
- scripts/hotel_normalization_robustness_commented.py -> hotel-wise normalized occupancy robustness analysis for both pooled EDA conclusions and ML comparisons
- reports/eda_summary.txt -> first EDA findings
- reports/modeling_summary.txt -> first baseline modeling findings
- reports/hotel_normalization_robustness_summary.md -> robustness summary generated by the hotel-wise normalization script
- reports/model_comparison.csv -> model-level evaluation metrics
- reports/model_comparison_by_hotel.csv -> hotel-level evaluation metrics
- model_outputs/hotel_normalization_robustness/ -> correlation tables, model comparisons, predictions, feature importance files, and robustness plots generated by the hotel-wise normalization script

Current result:
The lagged Google Trends random forest model outperformed the baseline-only random forest on the held-out time period.

Normalization note:
Because the project pools two hotels with different occupancy levels, the repository now includes a separate hotel-wise normalization robustness pass.
This robustness pass:
- recomputes pooled EDA correlations using hotel-normalized occupancy
- reruns the baseline vs baseline+trends ML comparison with a hotel-normalized target
- back-transforms predictions to raw occupancy units for interpretable RMSE comparison

Data note:
The robustness script auto-discovers the master table and best-lag correlation files in the repository, so it can be run even if the data folders are moved slightly.
